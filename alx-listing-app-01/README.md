# üè° ALX Listing App

## üåü Project Overview and Goals

The **ALX Listing App** project aims to scaffold and lay the foundational structure for a modern **Airbnb clone** property listing application.

This initial milestone focuses on setting up a well-organized and scalable codebase using **Next.js 13+**, **TypeScript**, **TailwindCSS**, and **ESLint**. By establishing a clean folder structure, reusable components, and adhering to best practices, the project ensures a solid starting point for building a dynamic, responsive, and user-friendly listing page.

---

## üèóÔ∏è Project Structure Explained

The project follows a standard pattern to promote modularity, reusability, and maintainability.

| Directory | Purpose | Example Files |
| :--- | :--- | :--- |
| **`components/`** | Houses all reusable UI components. | `components/common/Card.tsx`, `components/common/Button.tsx` |
| **`interfaces/`** | Stores all TypeScript type definitions and interfaces to ensure type safety. | `interfaces/index.ts` (for `CardProps`, `ButtonProps`, etc.) |
| **`constants/`** | Contains static, non-changing data used across the application. | `constants/index.ts` (for API URLs, configuration settings, or UI text) |
| **`public/assets/`** | Organizes static assets that are publicly accessible. | Placeholder images, SVGs, and other media files. |

### **Project Foundation Files**

* `components/common/Card.tsx`: Defines a reusable `Card` component to display information about properties.
* `components/common/Button.tsx`: Defines a reusable `Button` component for interactive actions (e.g., ‚ÄúBook Now‚Äù).
* `interfaces/index.ts`: Defines all core TypeScript interfaces (`CardProps`, `ButtonProps`, etc.) for type consistency.
